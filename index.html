Codex Workflow Toolkit

simple-codex

simple-codex는 프로젝트 내부에서 Codex CLI 사용 방식을 표준화하는 도구입니다. 프롬프트, 에이전트 규칙, 실행 순서를 구조화해 AI 협업을 재현 가능하고 예측 가능하게 만듭니다.

목적 핵심 개념 워크플로우 프롬프트/스킬 구조 설계 철학 로드맵

프로젝트 목적

매번 수동으로 프롬프트를 작성하는 대신, 팀이 동일한 규칙과 템플릿으로 Codex를 실행하도록 돕습니다. 이를 통해 개발 흐름의 일관성과 생산성을 높입니다.

핵심 개념: Architect / Executor

Architect Agent

설계와 계획을 담당합니다.

  • 요구사항 분석
  • 아키텍처/구현 전략 제시
  • 작업 단계 분해 및 변경 파일 정의
  • 직접 파일 수정은 수행하지 않음

Executor Agent

구현과 코드 수정을 담당합니다.

  • 코드 생성 및 수정
  • 프로젝트 규칙 준수
  • Architect 계획 기반 구현
  • 결과의 논리적 정합성 확인

표준 워크플로우

1) Setup

프로젝트를 Codex 사용 가능 상태로 초기화합니다.

simple-codex setup

2) Doctor

CLI 설치, 설정 파일, 경로 상태를 검증합니다.

simple-codex doctor

3) Prompt

역할별 사전 정의 프롬프트로 Codex를 실행합니다.

simple-codex prompt architect
simple-codex prompt executor

프롬프트 / 스킬 구조

Prompt Structure

역할 분리형 프롬프트 파일:

templates/codex/prompts/
├── architect.md
└── executor.md

Agent Skills

재사용 가능한 도메인 지식 및 워크플로우:

templates/agents/skills/
└── <skill-name>/
    └── SKILL.md

설계 철학

  • 구조화된 AI 워크플로우: 즉흥 프롬프트보다 정해진 단계 우선
  • 역할 분리: 설계와 구현 책임을 분리해 품질과 속도 균형 확보
  • 재현 가능성: 팀 전체가 같은 규칙으로 실행 가능
  • 간결한 CLI: 학습 비용을 낮추면서 확장 가능한 인터페이스 제공

로드맵 (Stage 1 ~ 4)

Stage 1 (완료)
  • setup / doctor / prompt 기반 템플릿 표준화
Stage 2 (현재)
  • 고정 역할 시퀀스 워크플로우 실행
  • 역할 간 출력 전달 (in-memory / temp 파일)
Stage 3
  • 워크플로우 상태 영속화(runId, step status, artifacts)
  • 재개/재실행/리플레이 및 단계별 재시도 제어
  • 최종 리포트 포맷(변경 요약, 에러, 다음 액션)
Stage 4
  • DAG 기반 의존성 워크플로우
  • 조건 분기(if/else) 및 자동 역할 선택
  • 안전한 병렬 실행과 결과 병합 전략
