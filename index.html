프로젝트 소개

simple-codex

simple-codex는 이 저장소에서 Codex CLI 사용 방식을 표준화해, 예측 가능하고 재현 가능한 AI 보조 개발 워크플로를 만드는 도구입니다.

공유 설정, 역할 기반 프롬프트, 재사용 가능한 에이전트 스킬, 환경 검증 명령을 하나의 흐름으로 제공합니다.

핵심 기능

공유 Codex 설정

프로젝트 범위에서 동일한 실행 환경을 맞춰 팀 간 결과 편차를 줄입니다.

역할 기반 프롬프트

architect, executor, review 역할별 책임을 분리해 출력 일관성을 높입니다.

스킬 + 환경 검증

재사용 가능한 스킬과 doctor 검증 흐름으로 실행 전후 상태를 빠르게 확인합니다.

사용 방법 (워크플로)

scodex workflow --scope project --task "<task>"

1. architect

요구사항 분석, 구현 계획 수립, 대상 파일 및 위험을 정의합니다.

2. executor

architect 출력을 받아 실제 파일 변경과 검증을 수행합니다.

3. review

architect와 executor 출력을 함께 검토해 품질과 요구사항 충족도를 평가합니다.

역할 정의

architect

  • 요구사항 분석 및 구현 가능한 설계안 제시
  • 모듈/파일 단위 분해 및 위험 식별
  • 파일 직접 수정은 수행하지 않음

executor

  • 설계 출력을 기반으로 범위 내 구현 진행
  • 작은 단위로 변경하고 검증 결과 보고
  • 불필요한 구조 변경 없이 요청 범위 유지

review

  • 출력 품질과 요구사항 커버리지 평가
  • 증거 기반 심각도(high/medium/low)로 이슈 보고
  • 직접 구현 패치는 제공하지 않음

로드맵

  • Stage 1 완료 setup, doctor, prompt template system
  • Stage 2 진행 중 fixed role sequence + output handoff
  • Stage 3 예정 persisted run state, resume/replay
  • Stage 4 예정 DAG workflow, branching, parallel safe steps
